# 計畫文件目錄 (PLAN)

本目錄存放所有模組的計畫文件（PLAN），定義「怎麼做」。

## 文件命名規範

- 格式：`[模組名稱].plan.md`
- 範例：`1.基本作業.plan.md`、`客戶基本資料維護.plan.md`

## 文件範本

每個計畫文件應包含：

1. **技術架構設計** - 如何實作此模組
2. **資料庫設計** - 資料表結構與關聯
3. **API 設計** - RESTful API 端點定義
4. **前端設計** - UI/UX 設計與元件規劃
5. **測試計畫** - 單元測試、整合測試、E2E 測試
6. **憲章遵循檢查** - 驗證是否遵守專案憲章所有原則

詳細範本請參考：[規格驅動開發 (SDD) 工作流程](../0.standards/3.規格驅動開發%20(SDD)%20工作流程.md)

## 狀態標記

每個計畫文件應標記狀態：
- `[草稿]` - 撰寫中
- `[審核中]` - 等待審核
- `[已核准]` - 已核准，可進入 TASKS 階段

#### 設計依據與總則
1. 要輸入帳號 及 密碼，經確認後，登入成功後，才可以進入海灣文件倉儲管理系統的首頁頁面。
2. 輸入密碼時應該遮蔽，用*號來代替顯示，另提供一個按鈕，可以開啟來顯示輸入的密碼是明碼。
3. 帳號或密碼錯誤時，統一提示：「您輸入的帳號或密碼不正確，請重新輸入。錯誤次數X次。」
4. 當錯誤超過 6次後，將該帳號鎖住，並跳出警告提示「您已連續輸入錯誤達6次，帳號已被鎖定。請與管理人員聯繫」的警告提示。
5. 帳號/密碼 驗證成功後，進入文件倉儲管理系統的首頁 Dashboard。

### 🔐 登入頁面設計與實作計畫

#### 第一階段：建立登入頁面組件
##### 1. 建立 Login 頁面 (src/pages/Login.tsx)
- 使用現有的 UI 組件系統（Card、Form、Input、Button）
- 整合 React Hook Form + Zod 進行表單驗證
- 響應式設計，適配各種螢幕尺寸
- 使用您的 Crown Logo 和品牌識別

##### 2. 表單功能設計
- 帳號輸入欄位：文字輸入框，必填驗證
- 密碼輸入欄位：
- 預設使用 type="password" 遮蔽顯示
- 添加眼睛圖示按鈕，可切換顯示/隐藏密碼
- 使用 Lucide React 的 Eye / EyeOff 圖示

##### 3. 驗證邏輯實作
- 測試帳號：預設一組管理員帳號進行驗證
- 錯誤追蹤：使用 localStorage 記錄每個帳號的錯誤次數
- 錯誤訊息：統一顯示「您輸入的帳號或密碼不正確，請重新輸入。錯誤次數X次。」
- 帳號鎖定：連續錯誤6次後鎖定，顯示警告對話框

##### 4. 警告對話框設計
- 使用現有的 AlertDialog 組件
- 顯示「您已連續輸入錯誤達6次，帳號已被鎖定。請與管理人員聯繫」
- 鎖定狀態持續保存在 localStorage

#### 第二階段：簡易認證狀態管理
##### 5. 建立基本 Auth Context (src/contexts/auth-context.tsx)
- 管理登入狀態 (isAuthenticated)
- 提供 login() 和 logout() 功能
- 使用 localStorage 保存登入狀態

##### 6. 建立 ProtectedRoute 組件
- 檢查用戶登入狀態
- 未登入時自動重導向到 /login
- 已登入時正常顯示頁面內容

#### 第三階段：路由整合
##### 7. 更新 App.tsx 路由結構
- 添加 /login 路由
- 將首頁和其他頁面包裝在 ProtectedRoute 中
- 登入成功後重導向到 Dashboard (首頁)

##### 8. 更新 Header 組件
- 添加登出功能按鈕
- 登出後清除狀態並重導向到登入頁

#### 第四階段：視覺設計優化
##### 9. 登入頁面視覺設計
- 居中的登入卡片設計
- 使用 Crown Logo 作為頁面頂部品牌標識
- 專業的表單佈局和間距
- 載入狀態指示器
- 一致的顏色主題和字體

##### 10. 響應式適配
- 手機端友好的佈局
- 平板和桌面的優化顯示
- 確保在各種解析度下都能正常使用

#### 技術實作細節
##### 使用的技術棧：
- React Hook Form + Zod (表單驗證)
- 現有 UI 組件系統 (Card, Form, Input, Button, AlertDialog)
- Lucide React (圖示)
- React Router (路由管理)
- localStorage (狀態持久化)

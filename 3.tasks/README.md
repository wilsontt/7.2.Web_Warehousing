# 任務文件目錄 (TASKS)

本目錄存放所有模組的任務文件（TASKS），將計畫拆解為可執行的任務。

## 文件命名規範

- 格式：`[模組名稱].tasks.md`
- 範例：`1.基本作業.tasks.md`、`客戶基本資料維護.tasks.md`

## 文件範本

每個任務文件應包含：

1. **任務清單** - 將計畫拆解為具體任務
2. **任務優先順序** - 標示任務的執行順序
3. **任務相依性** - 標示任務之間的依賴關係
4. **預估工時** - 每個任務的預估完成時間
5. **負責人** - 任務指派

每個任務應包含：
- 任務編號
- 任務描述
- 相關的 SPEC 使用者故事
- 驗收條件
- 預估工時

詳細範本請參考：[規格驅動開發 (SDD) 工作流程](../0.standards/3.規格驅動開發%20(SDD)%20工作流程.md)

## 狀態標記

每個任務應標記狀態：
- `[待處理]` - 尚未開始
- `[進行中]` - 執行中
- `[已完成]` - 已完成
- `[已驗收]` - 已驗收通過

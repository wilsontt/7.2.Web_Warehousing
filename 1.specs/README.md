# 規格文件目錄 (SPEC)

本目錄存放所有模組的規格文件（SPEC），定義「做什麼」。

## 文件命名規範

- 格式：`[模組名稱].spec.md`
- 範例：`1.基本作業.spec.md`、`客戶基本資料維護.spec.md`

## 文件範本

每個規格文件應包含：

1. **模組概述** - 描述此模組的核心目的
2. **功能清單與使用者故事** - 將功能轉化為使用者故事
3. **驗收條件** - 明確、可測試的驗收條件
4. **非功能性需求** - 效能、安全、語言等需求
5. **模組依賴性** - 描述此模組需要依賴哪些其他模組

詳細範本請參考：[規格驅動開發 (SDD) 工作流程](../0.standards/3.規格驅動開發%20(SDD)%20工作流程.md)

## 狀態標記

每個規格文件應標記狀態：
- `[草稿]` - 撰寫中
- `[審核中]` - 等待審核
- `[已核准]` - 已核准，可進入 PLAN 階段

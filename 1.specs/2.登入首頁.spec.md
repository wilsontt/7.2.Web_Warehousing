## 1. 目的 (Purpose)
本模組旨在提供系統登入後的主要操作介面（Dashboard），包含全域導覽列、功能選單以及即時資訊看板，讓使用者能快速掌握工作狀態並進入各項作業功能。

## 2. 範圍 (Scope)
- **全域佈局 (Global Layout)**: 包含頂部導覽列 (Header) 與主要內容區 (Main Content)。
- **頂部導覽列 (Header)**:
    - 品牌識別 (Logo + Title)。
    - 主選單 (Main Menu): 8 大功能模組的下拉選單，需包含圖示與編號。
    - 使用者工具列: 使用者選單 (包含語系切換、深淺色模式切換)。
- **首頁看板 (Dashboard Widgets)**:
    - 上半部：今日任務、庫存概況、最新通知。
    - 下半部：倉庫詳細資訊、作業狀態、排行榜、趨勢圖。

## 3. 使用者故事 (User Stories)

### US-001: 全域導覽
**作為** 系統使用者
**我想要** 在任何頁面都能透過頂部導覽列切換功能
**以便於** 快速進入不同的作業模組

**驗收條件**:
1.  **固定置頂**: 導覽列應固定在頁面頂部 (Sticky)。
2.  **主選單**: 顯示 8 大功能模組（基本、例行、庫存、客戶、移儲、稽核、倉儲管理、系統管理）。
    *   **樣式要求**: 每個模組名稱前需加上對應的 **Icon** 與 **編號** (如：1. 基本作業)。
3.  **下拉選單**: 滑鼠移入或點擊主選單項目時，應展開對應的子功能列表。
    *   **基本作業**: 代碼作業 (基本代碼, 箱件代碼), 員工資料, 客戶資料 (基本資料, 客戶群), 倉庫車輛 (倉庫, 車輛, 收送區域), 箱子資料, 基本收費。
    *   **例行作業**: 客戶工作通知單, 倉儲工作單, 倉儲作業 (派車, 上下架/稽核報表, 工作單輸入), 出倉車輛運輸單, 交換作業 (玉山, 中信)。
    *   **庫存作業**: 即時庫存查詢, 倉儲報表 (工作報表, 即時庫存, 月報, 統計表, 進出月報, 物件清單, 運輸單), 條碼作業 (箱子, 物件, 儲位, 封箱, 新光特規)。
    *   **客戶作業**: 客戶工作通知單, 審核作業, 即時庫存查詢, 倉儲報表下載。
    *   **移儲作業**: 移出作業 (移出單, 下架, 稽核), 移入作業 (移入單, 下架, 稽核)。
    *   **稽核作業**: 抽查記錄表 (A003, A004, A005), 盤點作業 (盤點單, 初盤, 複盤)。
    *   **倉儲管理作業**: 工作單維護, 移儲單維護, 盤點單維護, 主管審核 (通知單取消, 移出入取消, 盤點取消), 庫存維護。
    *   **系統管理作業**: 帳號設定, 參數設定 (通知, 多語系), 客戶使用者, 角色設定, 權限控管 (群組權限, 使用者群組, 功能選單), 資料交換, 郵件通知, 最新訊息。
4.  **權限過濾**: 應根據使用者角色，僅顯示有權限的選單項目 (Phase 1 先實作 UI，權限邏輯預留)。

### US-002: 使用者個人化設定
**作為** 系統使用者
**我想要** 切換語系或深淺色模式
**以便於** 調整介面以符合我的閱讀習慣

**驗收條件**:
1.  **使用者選單**: 顯示登入者名稱，下拉可見：
    *   使用者資訊 (帳號、IP)。
    *   **語系切換**: 提供 中文/English 切換選項。
    *   **深淺色切換**: 提供切換按鈕。
    *   **字體大小**: 提供 [小]、[中]、[大] 三段調整，需連動全站縮放。
    *   變更密碼。
    *   登出選項。
2.  **位置調整**: 語系與深淺色切換功能應整合至使用者下拉選單中，置於「變更密碼」上方，不單獨顯示於 Header 工具列。

### US-003: 首頁資訊看板 (Dashboard)
**作為** 倉儲管理人員
**我想要** 在登入後立即看到關鍵營運數據
**以便於** 掌握今日工作重點與異常狀況

**驗收條件**:
1.  **今日任務卡片**: 顯示未完成單據、待出貨數、待稽核數。
2.  **庫存概況卡片**: 顯示總箱數、使用率。
3.  **最新通知卡片**: 顯示最新公告或異常提醒。
4.  **倉庫資訊區**: 顯示各倉庫的存放/借出數量、今日工作狀態。
5.  **圖表呈現**: 顯示逾時未銷毀 Top 10、存倉數量 Top 10 (使用長條圖或列表呈現)。

## 4. 介面設計 (UI Design)
- **風格**: 參考 `https://femashr.com/` 的簡潔科技感。
- **配色**: 淺色底、淡水藍色線條、圓角按鈕。
- **響應式**: 平板與桌面優先，手機端選單可折疊為漢堡選單。

## 5. 技術與資料需求
- **圖表庫**: 建議使用 `recharts` 或 `chart.js` 實作圖表。
- **Icons**: 使用 `lucide-react`。
- **Layout**: 使用 CSS Grid 或 Flexbox 實作 A 區 (Header) + B 區 (Content) 佈局。

# 聊天記錄 (Chat History)

## 目的

本資料夾用於儲存專案開發過程中的重要對話記錄，包括：
- 設計決策討論
- 技術選型討論
- 問題解決過程
- 程式碼審閱意見
- 架構設計討論

## 檔案命名規範

建議使用以下命名格式：

```
YYYY-MM-DD-{主題}-{序號}.md
```

**範例**:
- `2025-11-28-共用元件設計討論.md`
- `2025-11-28-代碼維護功能實作-01.md`
- `2025-11-29-資料庫結構討論.md`

## 內容結構建議

每份聊天記錄檔案建議包含以下章節：

1. **日期與時間**: 對話發生的日期時間
2. **參與者**: 參與討論的人員（如：Wilson Tzou, AI Assistant）
3. **主題**: 討論的核心主題
4. **討論內容**: 主要對話內容
5. **決策摘要**: 最終的決策與結論
6. **後續行動**: 需要跟進的事項

## 使用方式

### 方式一：手動複製貼上
1. 在 Cursor 中完成重要討論後
2. 複製對話內容
3. 建立新檔案並貼上內容
4. 進行必要的整理與編輯

### 方式二：請 AI Assistant 自動生成
在 Agent 模式下，可以直接請 AI Assistant 將討論內容總結並寫入檔案：

```
請將剛才關於 [主題] 的討論，總結並寫入到 
docs/chat_history/YYYY-MM-DD-{主題}.md
```

## 注意事項

- 這些記錄會納入 Git 版本控制，請確保不包含敏感資訊（如 API 金鑰、密碼等）
- 定期整理，移除過時或重複的內容
- 重要決策建議同時更新到相關的規格文件或設計文件中

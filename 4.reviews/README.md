# 驗收文件目錄 (REVIEW)

本目錄存放所有模組的驗收文件（REVIEW），記錄驗收結果與測試報告。

## 文件命名規範

- 格式：`[模組名稱].review.md`
- 範例：`1.基本作業.review.md`、`客戶基本資料維護.review.md`

## 文件範本

每個驗收文件應包含：

1. **功能驗收清單** - 檢查所有功能是否符合規格
2. **測試報告** - 單元測試、整合測試、E2E 測試結果
3. **效能測試報告** - 效能指標是否符合需求
4. **安全測試報告** - 安全性檢查結果
5. **主題一致性檢查** - 深/淺色主題切換測試
6. **問題清單** - 發現的問題與修正狀態
7. **驗收結論** - 是否通過驗收

詳細範本請參考：[規格驅動開發 (SDD) 工作流程](../0.standards/3.規格驅動開發%20(SDD)%20工作流程.md)

## 狀態標記

每個驗收文件應標記狀態：
- `[驗收中]` - 正在進行驗收
- `[待修正]` - 發現問題，待修正
- `[已通過]` - 驗收通過
- `[已上線]` - 已部署至生產環境
